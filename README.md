## 📌 Proyectos destacados

### FanCollector 🗂️
[API](https://github.com/NereaTM/FanCollector-API) · [Frontend React](https://github.com/NereaTM/FanCollector-React) · [Android](https://github.com/NereaTM/FanCollector-Android) · [Mock](https://github.com/NereaTM/FanCollector-mock)

Plataforma para gestionar colecciones. El proyecto ha evolucionado en dos versiones: la primera incluye una app Android y un mock server que simulaba el backend para poder desarrollar el cliente sin depender de la API real; la segunda reemplaza eso por un frontend en React + TypeScript contra la API  en Spring Boot. Uso CI/CD con GitHub Actions para mantener la aplicación desplegada en un clúster de AWS EKS, y Newman se encarga de ejecutar los tests de integración en cada PR para asegurarse de que nada se rompe antes de llegar a producción.

### [Practica EKS + CI/CD](https://github.com/NereaTM/Practica-Eks-CiCd) ☁️
Práctica centrada en entender qué ocurre después del git push: la imagen se construye, se publica y se despliega sola en Kubernetes. El objetivo era tener un flujo real de despliegue continuo, no solo configurarlo en local.

### [Discord API – NestJS](https://github.com/NereaTM/Discord-Api-Nest) 🤖
Integración con la API de Discord en NestJS y TypeScript. Una excusa para explorar una arquitectura distinta a Spring fuera del ecosistema Java.

### [Web Bar – Dados y Tragos](https://github.com/NereaTM/Web-Bar-DadosYTragos) 🃏
Aplicación web en Java con JSP y Servlets para la gestión de un bar temático. Fue uno de los primeros proyectos donde construí una aplicación web clásica de principio a fin con Maven y Tomcat.

### [PixelPanda – Anime Database](https://github.com/NereaTM/PixelPanda-dbAnime) 🐼
Frontend en HTML, CSS y JavaScript que consume la API de Jikan para explorar anime. Proyecto centrado en la experiencia de usuario y en aprender a trabajar con APIs externas desde el cliente.
